The primary way to adapt the FHIR specification to specific requirements of a jurisdiction is through extensions.

Extensions published by HL7 Finland

This list includes the extensions for scheduling related profiles defined by HL7 Finland.

  • Additional Information URL

    URL for additional information regarding appointment

  • Ajanvarauksen tila

    Binding of AppointmentStatus to the Finnish logical model Ajanvarauksen tila.

  • Care Guarantee Details

    An extension for including care guarantee (hoitotakuu) details in appointment.

  • Careplan Identifier

    Identifier for the care plan, if provided.

  • Child Appointment

    Reference from a master (parent) appointment to its sub (child) appointments

  • Customer Journey

    Identifier for the customer journey (hoitopolku)

  • Appointment Mutability

    Extension specifying whether appointment can be changed and when.

  • Notification Information

    Information on selected notification method and address

  • Notification Medium

    Information on notification medium, based on THL - Viestintäkanava codesystem.

  • Patient Instruction URL

    URL for patient instruction.

  • Practitioner Gender

    Extension for practitioner gender for appointment.

  • Queue Number

    An extension for queue number (in string format).

  • Referral Id

    Referral id extension for appointment (string)

  • Requested Service Info

    Extension for providing additional information for appointment service: reserved product/service.

  • Resource Calendar

    Extension for resource calendar id and resource name.

  • Self Service Information

    Extension that notifies whether there is a self-service for managing appointment (provides URL and information on whether management can be made by proxy).

  • Additional Information on Service

    Additional details regarding the service the appointment concerns.

  • Ticket Number

    Ticket number to be included in the appointment.

  • Topic Id

    Social care topic (asia) id.

Many of these extensions have been defined in earlier implementation guides. The canonical URLs of the extensions have been modified slightly, mainly to be in the https space, rather than http. Also the names of the extensions have been changed to be more descriptive. Implementer feedback is welcome at this stage to help evaluate whether this is a welcome change, and regarding implementation efforts.

The earlier versions of these extensions can be found at FinnishAppoinment and FinnishScheduling R4 implementation guides.

Some of the extensions have been moved to the Finnish Base Profiles specification. These include extensions related to service event (palvelutapahtuma) and to visibility of information.

Extensions in other Finnish Implementation Guides

In addition to the extensions published by HL7 Finland, there are also other extensions used in Finland. Most notably the ones for the national Kanta system, in several implementation guides:

  • Kanta yhteiset FHIR- ja REST-rajapintamäärittelyt includes definitions shared by several other Kanta implementation guides.
  • Kanta Potilastiedon arkiston FHIR R4 included specifications for patient data stored in Kanta.
  • Kanta FHIR Prescription R4 includes specifications for data on prescriptions
  • Kanta sosiaalihuolto R4 includes specifications for data related to welfare and social affairs.
  • Finnish PHR R4 includes specifications for health and wellbeing data curated by the citizens in the Finnish national Personal Health Record platform Kanta PHR.
  • Kanta tietojen kansalaiskäytön rajapinnat
    includes specifications for citizen-centric apps accessing data in Kanta.

Other Extensions

In addition to above, a registry of standard extensions can be found in the FHIR specification and additional extensions may be registered on the HL7 FHIR registry at http://hl7.org/fhir/registry.
