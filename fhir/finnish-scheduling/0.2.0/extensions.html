The primary way to adapt the FHIR specification to specific requirements of a jurisdiction is through extensions.

Extensions for Finnish Base Profiles

This list includes the extensions for scheduling related profiles defined by HL7 Finland.

  • ParentAppointment

    Master appointment reference using identifier type Pääajanvarauksen tunniste

  • ChildAppointment ChildAppointment
  • MiscAppointmentDataExtension

    Other details regarding appointment. 91 Muu sisältö

  • NotificationInfoExtension

    Information on selected notification method and address

  • AdditionalInformationURLExtension

    URL for additional information regarding appointment 98 Lisätietojen osoite

  • AppointmentMutabilityExtension

    Extension for FinnishAppointment specifying whether appointment can be changed and when.

  • CareGuaranteeExtension

    An extension for including care guarantee (hoitotakuu) details in appointment.

  • CareplanIdentifierExtension

    Identifier for the care plan, if provided.

  • CustomerJourneyExtension

    Identifier for the customer journey (hoitopolku) 20 Asiakaspolun tunniste 20.1 Asiakaspolun nimi

  • HomeMunicipalityExtension

    An extension for patient home municipality.

  • NotificationMediumExtension

    Information on notification medium, based on THL - Viestintäkanava codesystem 94 Muistutuksen tapa

  • PractitionerGenderExtension

    Extension for practitioner gender for appointment. 93 Palvelun toteuttajan sukupuoli

  • QueueNoExtension

    An extension for queue number (in string format). 82 Jonotusnumero

  • ReferralIdExtension

    Referral id extension for appointment (string). 24 Ajanvaraukseen liittyvän lähetteen tai pyynnön tunniste

  • RequestedServiceExtension

    Extension for providing additional information for appointment service: reserved product/service. 73 Varaustuote 73.1 Varaustuotteen nimi

  • ResourceCalendarIdExtension

    Extension for resource calendar id and resource name. 74 Resurssin kalenteritunniste. 75 Resurssin nimi

  • SelfServiceExtension

    Extension for Finnish Scheduling appointment, that notifies whether there is a self service for managing appointment (provides URL and information whether management can be made by proxy). 84 AsiointiURL 84.1 Järjestelmä tukee alaikäisen puolesta asiointia 84.2 Järjestelmä tukee puolesta-asiointia valtakirjalla täysi-ikäisen puolesta 84.3 Järjestelmä tukee rekisteripohjaista puolesta-asiointia täysi-ikäisen puolesta

  • ServiceEventExtension

    Extension for service event id (palvelutapahtuman tunniste). 22 Palvelutapahtuman tunniste

  • TicketNoExtension

    Ticket number to be included in the appointment. 101 Tiketti

  • TopicIdExtension

    Social care topic (asia) id. 23 Asian tunniste

  • VisibleToCustodianExtension

    59 Ajanvarauksen tietojen näyttäminen huoltajalle

    1=Ajanvarauksen tiedot saa näyttää huoltajalle,2=Ajanvarauksen tietoja ei saa näyttää huoltajalle

  • PatientInstructionURLExtension

    URL for patient instruction. Linkki potilasohjeeseen

Extensions in other Finnish Implementation Guides

Please also note the extensions listed in the Finnish Base Profiles specification.

In addition to the extensions published by HL7 Finland, there are also other extensions used in Finland. Most notably the ones for the national Kanta system, in several implementation guides:

  • Kanta yhteiset FHIR- ja REST-rajapintamäärittelyt includes definitions shared by several other Kanta implementation guides.
  • Kanta Potilastiedon arkiston FHIR R4 included specifications for patient data stored in Kanta.
  • Kanta FHIR Prescription R4 includes specifications for data on prescriptions
  • Kanta sosiaalihuolto R4 includes specifications for data related to welfare and social affairs.
  • Finnish PHR R4 includes specifications for health and wellbeing data curated by the citizens in the Finnish national Personal Health Record platform Kanta PHR.
  • Kanta tietojen kansalaiskäytön rajapinnat
    includes specifications for citizen-centric apps accessing data in Kanta.

Other Extensions

In addition to above, a registry of standard extensions can be found in the FHIR specification and additional extensions may be registered on the HL7 FHIR registry at http://hl7.org/fhir/registry.
